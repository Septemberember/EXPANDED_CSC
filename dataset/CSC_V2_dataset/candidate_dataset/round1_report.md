# Candidate Dataset Round 1 Report

Round 1 generated two original Java subjects and six single-fault mutants.

## Subjects

### ShippingCost

Original:

- `ShippingCost/ShippingCost.java`

Target method:

- `public static int calculateShipping(int weight, int distance, boolean express)`

Program features:

- invalid input guards,
- business thresholds for weight and distance,
- bounded loop over extra weight,
- bounded loop over distance bands,
- express surcharge,
- return-value clamp.

Mutants:

- `ShippingCost_M1.java`: BOR, `distance > 500` to `distance >= 500`
- `ShippingCost_M2.java`: CR, express surcharge `10` to `8`
- `ShippingCost_M3.java`: AOR, distance band update `cost += 4` to `cost -= 4`

Smoke status:

- Original passed `csc_tool.py` with `--max-iter 2`.
- All three mutants passed `csc_tool.py` with `--max-iter 2`.
- `ShippingCost_M3` also passed a legal-input bootstrap smoke test with
  `weight=3,distance=250,express=true`, exercising loop instrumentation.

### DiscountCalculator

Original:

- `DiscountCalculator/DiscountCalculator.java`

Target method:

- `public static int computeDiscount(int price, int customerYears, boolean coupon)`

Program features:

- invalid input guards,
- price threshold branches,
- bounded loop over customer years,
- coupon condition,
- return-value clamp.

Mutants:

- `DiscountCalculator_M1.java`: BOR, `price >= 500` to `price > 500`
- `DiscountCalculator_M2.java`: LOR, `coupon && price >= 100` to
  `coupon || price >= 100`
- `DiscountCalculator_M3.java`: SUR, `yearsLeft -= 5` to `yearsLeft -= 4`

Smoke status:

- Original passed `csc_tool.py` with `--max-iter 2`.
- All three mutants passed `csc_tool.py` with `--max-iter 2`.
- `DiscountCalculator_M3` also passed a legal-input bootstrap smoke test with
  `price=500,customerYears=12,coupon=true`, exercising loop instrumentation.

## Notes

The default bootstrap input generated by `csc_tool.py` is often zero-valued.
For subjects with invalid-input guards, a short `--max-iter 2` smoke test may
mostly exercise guard paths. A second targeted bootstrap with valid inputs is
useful for checking deeper business logic and loops.

Round 1 did not run full refined TBFV or kill/survival classification. The
purpose of this round was to verify that generated originals and mutants can be
instrumented, compiled, executed, traced, and added to a CCT.
